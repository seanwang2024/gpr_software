# 劳雷探地雷达处理软件算法说明

本文档描述本软件中 11 个数据处理功能涉及的算法原理与实现要点。
所有功能均针对 DZT 格式数据（32 位有符号整数小端采样，512 采样点/道），
输入数据已在显示满量程 ±2^23（±8388608）范围内（LUT 映射 `value/65536+128`）。

---

## 一、零点调节

**位置**：菜单 开始 → 简易处理 → 调节零点

### 算法
基于"时间位置零点"百分比（对话框中可调，默认值由用户设置），将图像的零点位置向上平移：
- 设百分比为 `p%`（典型 -2%），对应时间偏移 `dt = -p × 0.2`（即 -p×4 ns 在 20 ns 全量程下）
- 跳过的行数：`skip = round(samplesPerTrace × |dt| / timeRange)`，512 采样时 `p=2` → 跳过约 51 行
- 应用时记录 `zeroSkipRows`，在 `refreshImage` 中从第 `skip` 行开始读取（等效时间零点下移）
- 不修改底层 `rawData`，仅改变显示窗口起点；与底层处理后数据相互独立

### 特点
- 纯显示层操作，可与其他处理叠加
- 重设即恢复原始时间零点

## 二、调节增益

**位置**：菜单 开始 → 简易处理 → 调节增益；菜单 数据处理 → 滤波组 → 调节增益

### 算法
对每条 trace 沿深度方向（采样点 s = 0..511）施加时变增益：
- 用户在 CustomChartView 中拖拽 8 个增益手柄（折线），构成 gain(s) 函数
- 每个采样点位置 s 的增益值通过对各手柄分段线性插值得到
- 增益类型可选：
  - **dB**（默认）：`factor(s) = 10^(gain_db(s) / 20)`
  - **线性**：`factor(s) = gain_linear(s)`
- 应用：`out[i] = clamp(factor(i mod 512) × in[i], ±2^23)`，clamp 防止溢出

### 特点
- 时变增益补偿球面扩散衰减，提升深部信号可视性
- 拖拽手柄实时预览，"应用"将增益乘入 `rawData`
- "撤销"恢复 `originalRawData`

## 三、校正零偏 (Dewow)

**位置**：菜单 开始 → 简易处理 → 校正零偏；菜单 数据处理 → 滤波组 → 校正零偏

### 算法
对每条 trace 沿深度方向做滑动窗口均值去除（去除低频"wow"漂移）：
- 输入参数：时间窗口 `timeWindowNs`（默认 40 ns）、天线频率 `antennaFreqMHz`（默认 900 MHz，仅参考用）
- 计算每采样时间间隔 `dt = timeRange / samplesPerTrace`
- 窗口采样数 `W = round(timeWindowNs / dt)`，半窗 `halfWin = W/2`
- 对每个 trace 独立计算前缀和 `cumsum`，O(1) 求每个采样点 s 的局部均值：
  `localMean(s) = (cumsum[s+halfWin+1] - cumsum[s-halfWin]) / (2·halfWin+1)`
- 输出 `out[s] = in[s] - localMean(s)`，边界截断

### 特点
- 等效于时变低频抑制（DC + 极低频去除）
- 前缀和实现 O(N) 复杂度，对每条 trace 独立处理
- 保留相对直流偏置去除后的高频反射特征

## 四、背景消除

**位置**：菜单 开始 → 简易处理 → 背景消除；菜单 数据处理 → 滤波组 → 背景消除

### 算法
沿水平方向（trace 方向）消除背景，去除水平相干噪声（如直达波、地表耦合波）：

| 模式 | 方法 |
|---|---|
| **整体法** | 全局背景：`bg(s) = mean(in[t, s] over all t)`，每点减去对应 `bg(s)` |
| **扫描范围** | 滑动窗口背景：`bg(t, s) = mean(in[tt, s] for tt in [t-halfWin, t+halfWin])`，窗口大小默认 200 道 |
| **全部通过** | 与整体法相同（等价于无背景保留） |
| **自适应** | 高斯加权滑动背景：权重 `w(tt) = exp(-(tt-t)² / (2σ²))`，σ = halfWin/3，halfWin = numTraces/20 |
| **无** | 跳过处理 |

### 特点
- 整体法/扫描范围适用于水平层状背景噪声
- 自适应法在背景非平稳时表现更好（边缘不被过消除）
- 不修改深部数据极性

## 五、数字滤波

**位置**：菜单 开始 → 简易处理 → 数字滤波；菜单 数据处理 → 滤波组 → 数字滤波

### 算法
沿深度方向（每条 trace）应用频率域滤波，支持四种带型 × 两种实现：

**带型**（基于低频 `lowHz`、高频 `highHz`）：
- 低通、高通、带通、带阻

**实现类型**：
1. **FIR（默认，32 阶）**
   - 用理想滤波器频响的逆傅里叶得到 `hd[n]`（sinc 函数族）
   - 加 Hamming 窗 `w[n] = 0.54 - 0.46·cos(2πn/M)`
   - 时域卷积输出（边界截断）

2. **IIR Butterworth（8 阶，前向-后向双向滤波 → 等效 16 阶）**
   - 双线性变换预扭曲：`Wc = tan(π·fc/fs)`
   - 分解为 4 个二阶节（SOS, Biquad）级联
   - 带通 = 高通(lowHz) ∘ 低通(highHz)，带阻反之
   - 每节用 Direct Form II 转置结构（w0 = x - a1·w1 - a2·w2, y = b0·w0 + b1·w1 + b2·w2）
   - 前向+后向双向滤波实现零相位

### 特点
- FIR 相位线性，但过渡带较宽
- IIR 零相位（双向），过渡带陡峭
- 对话框实时显示处理前/后频谱（FFT 后取 dB，归一化到自身峰值）

## 六、滑动平均

**位置**：菜单 数据处理 → 滤波组 → 滑动平均

### 算法
对每条 trace 沿深度方向做滑动平均（一维 boxcar 平滑）：
- 输入参数：窗口大小 `winSize`（默认 8）
- 半窗 `halfWin = winSize/2`
- 对每个采样点 s：`out[s] = mean(in[s-halfWin .. s+halfWin])`，边界截断
- 仅沿深度方向（s 轴），不跨 trace

### 特点
- 等效于简单的低通滤波，压制高频随机噪声
- 与"一键处理"中的滑动平均步骤使用同一算法，保证结果一致

---

## 七、道间均衡 (Trace Equalization)

**位置**：菜单 数据处理 → 滤波组 → 道间均衡

### 算法
基于 RMS 振幅的道间能量均衡：
1. 计算每条 trace t 的 RMS 振幅：
   `rms(t) = sqrt(mean(in[t, s]² over s))`
2. 参考值为所有 trace RMS 的平均：
   `rms_ref = mean(rms(t) over t)`
3. 缩放因子：
   `scale(t) = rms_ref / rms(t)`（rms(t) 极小时保持原值，scale = 1）
4. 输出：`out[t, s] = round(in[t, s] × scale(t))`

### 特点
- 平衡各 trace 间的能量差异（如天线耦合不均、衰减差异）
- 保留每条 trace 内部的相对时变特征
- 仅调整 trace 间整体幅度，不改变波形

## 八、数学运算

**位置**：菜单 数据处理 → 其他处理 → 数学运算

### 算法
对每个采样点应用逐点数学运算（沿深度方向计算），6 种运算类型：

| 类型 | 公式 |
|---|---|
| **差分** | 中心差分 `out[s] = (in[s+1] - in[s-1]) / 2`；边界单侧差分 |
| **积分** | 沿 s 方向累加：`out[s] = sum(in[0..s])` |
| **平方** | `out[s] = in[s]²` |
| **开方** | 保留符号：`out[s] = sign(in[s]) × sqrt(|in[s]|)` |
| **对数** | 保留符号：`out[s] = sign(in[s]) × ln(|in[s]|)`（0 时输出 0） |
| **指数** | 保留符号，幅值裁剪到 ±50 防爆炸：`out[s] = sign(in[s]) × exp(in[s]/srcMax)` |

**归一化**（可选）：
- "是"：线性映射到原始 `[srcMin, srcMax]` 范围
- "否"：直接按 qint32 范围裁剪

### 特点
- 差分/积分用于强调突变/累积特征
- 平方/对数改变动态范围，便于显示微弱反射
- 归一化保证显示一致性

## 九、反褶积 (Deconvolution)

**位置**：菜单 数据处理 → 其他处理 → 反褶积

### 算法
RADAN 风格的 Wiener 预测反褶积（每条 trace 独立处理）：

1. **自相关**：计算 trace 自相关序列
   `r[k] = mean(in[n]·in[n+k])` for k = 0..L+p-1

2. **构造 Wiener-Hopf 方程**：
   - Toeplitz 矩阵 `R[i][j] = r[|i-j|]`
   - 期望响应 `g[i] = r[predStep+i]`
   - 白噪声加成 `r[0] *= 1.001` 保证正定

3. **高斯消元（部分主元）求解**：`R·f = g`，得到预测滤波器 `f[k], k=0..L-1`

4. **预测误差输出**（残差 = 反褶积结果）：
   `e[n] = in[n] - sum(f[k]·in[n-predStep-k] for k=0..L-1)`

**归一化**：将残差线性映射回原始 `[srcMin, srcMax]` 范围。

### 参数
- 滤波器长度 `filterLen`（默认 31）
- 预测步长 `predStep`（默认 5）：等于子波长度时压制多次波

### 特点
- 压制多次波、压缩子波（提升分辨率）
- 预测步长决定被压制的多次波周期
- Toeplitz 结构可专用于 Levinson 算法（当前实现用通用高斯消元，鲁棒但稍慢）

## 十、希尔伯特变换 (Hilbert Transform)

**位置**：菜单 数据处理 → 其他处理 → 希尔伯特变换

### 算法
通过 FFT 计算解析信号 `z[n] = in[n] + j·H{in}[n]`，再派生三种属性：

1. **解析信号构造**：
   - FFT 长度 `fftN` = ≥samplesPerTrace 的最小 2 的幂
   - 实信号频谱 `X[k] = FFT(in[n])`
   - 解析信号谱：
     - `Z[0] = X[0]`
     - `Z[k] = 2·X[k]` for k = 1..fftN/2-1（正频加倍）
     - `Z[k] = 0` for k = fftN/2+1..fftN-1（负频置零）
   - `z[n] = IFFT(Z[k])`

2. **三种输出**：
   - **瞬时振幅**（包络）：`|z[n]|`
   - **瞬时相位**：`arg(z[n])`（弧度，×1000 缩放）
   - **瞬时频率**：相位解卷绕后中心差分
     - 解卷绕：`dφ[n] = unwrap(φ[n] - φ[n-1]) + φ[n-1]`
     - 频率：`f[n] = (φ[n+1] - φ[n-1]) / (4π)` (cycle/sample)，×1000 显示，限制在 [0, 0.5]

**归一化**：所有输出线性映射回原始 `[srcMin, srcMax]` 范围。

### 特点
- 瞬时振幅突出反射强度（地层界面）
- 瞬时相位突出地层连续性（不依赖振幅大小）
- 瞬时频率反映岩性变化

## 十一、克希霍夫迁移 (Kirchhoff Migration)

**位置**：菜单 数据处理 → 其他处理 → 克希霍夫迁移

### 算法
时间域 Kirchhoff 求和偏移，将绕射双曲线收敛到绕射源：

**输入参数**：
- `firstWave`：首波采样位置（默认 27）
- `vel`：等效偏移速度 `v_mig`（默认 0.106 m/ns，业界惯例已含发收双程因子）
- `aperture`：最大偏移孔径（道数，默认 60）
- `spacing`：道间距（cm，默认 1.0）

**预处理**：
- DC 去除：减去整张图像所有采样的中位数
  （仪器偏移使深部整体偏置 > 0；中位数对绕射峰鲁棒，避免减去有效信号）

**核心偏移公式**（时间域 Kirchhoff 积分）：

对输出位置 `(x_out, τ0)`：
1. 计算零偏移双程走时：`τ0 = (s_out - firstWave) × tps`，对应深度 `z = v_mig·τ0/2`
2. **动态孔径**：限制最大倾角 θ_max=45°，`dx_max = z·tan(45°) = z`
   - 浅层 z 小 → 孔径自动收缩到 1 道 → 不被远道反向极性抵消
   - 深部 z 大 → 孔径自动放大（上限 aperture/2）
3. **绕射双曲线**：对孔径内每个输入道 x_in：
   - 水平偏移 `dx = (x_in - x_out) × dx_m`
   - 走时 `τ_in = sqrt(τ0² + (dx/v_mig)²)`
   - 采样位置 `s_in = firstWave + τ_in/tps`，线性插值取值 `val(x_in, s_in)`
4. **Yilmaz 标准权重**（倾斜因子 × 反混叠因子）：
   `w = (τ0/τ_in)^(3/2) = cos(θ)·sqrt(τ0/τ_in)`
5. **加权平均**：`out(x_out, s_out) = Σ(w·val) / Σw`

**输出**：
- 首波之上（τ0 ≤ 0）：直接保留 DC 去除后的原值
- 写回 `rawData`，仅 qint32 范围裁剪，不做后处理缩放

### 双曲线可视化（绿色叠加）
鼠标悬停在图像上时，绘制以鼠标位置为顶点的绿色双曲线（绕射拟合辅助）：
- 公式：`τ(dx) = sqrt(τ0² + (dx/v_mig)²)`（与偏移算法一致）
- 实时受对话框参数影响（首波、速度、孔径、道间距）
- 用于交互式估计绕射源位置和速度参数

### 特点
- 动态孔径避免浅层被远道反向极性抵消
- Yilmaz 权重软截断远道拖尾，锐化聚焦
- 归一化加权平均保留相干绕射峰振幅
- 输入已 ±2^23 满量程，无需后处理缩放（任何额外缩放都会破坏显示量程）

## 通用设计要点

### 应用/撤销模式
所有处理对话框采用统一的"应用/撤销"模式：
- **应用**：备份 `originalRawData` → 处理 → 写入 `rawData` → 按钮变"撤销"
- **撤销**：恢复 `originalRawData` → 按钮变"应用"
- **确定**：应用后调用 `saveProcessedFile()`，生成 `proc/<原文件名>_p##.DZT` 新文件并以新 tab 打开
- **取消**：若已应用则还原，再关闭对话框

### 数据格式
- DZT 文件：32 位有符号整数小端，512 采样/道
- 显示 LUT 映射：`idx = value/65536 + 128`，输入值域 ±2^23 对应 idx 0..255
- 钳位：所有输出限制在 ±2^23（防止 qint32 溢出显示范围）

### 处理范围
"数据处理 → 处理范围"组（起始道/终止道/起始点/终止点）允许用户限定处理作用区域；
当前各处理函数对全数据生效，处理范围 spinbox 用于显示参考。
